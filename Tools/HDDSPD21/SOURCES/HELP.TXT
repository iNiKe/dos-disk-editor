.topic MainIndex=7

 HddSpeed ▄
  ▀▀▀▀▀▀▀▀▀

   HddSpeed  -  это  универсальная  программа для измерения скоростных
характеристик  жестких  дисков и диагностики их состояния.

   Тесты  скоростных характеристик, используемые в программе, наиболее
полно отражают быстродействие диска при различных операциях. С помощью
HddSpeed   была   собрана   обширная   база  результатов  тестирования
быстродействия   более   чем  200  накопителей,  что  позволяет  легко
сравнивать получаемые результаты. Измеряемые характеристики достаточно
просты   и,   в   то же время, реалистично отражают скорость диска при
повседневной работе.

   Уникальные  методы детального тестирования жестких дисков позволяют
производить  диагностику  состояния  электроники накопителя и качество
магнитной  поверхности.  По  виду  графика  скорости  чтения  по  всем
дорожкам  диска  можно  сделать  выводы  о  наличии  плохих  блоков, и
замещенных   дорожек.   В   отличие  от  других  утилит,  использующих
низкоуровневые  методы  и  закрытую информацию по работе с замещенными
дорожками,    данный    метод    универсален    и    не   зависит   от
фирмы-производителя  и  модели  накопителя, поэтому может быть успешно
использован на любых дисках.

.topic DiskSelect

 Select ▄
  ▀▀▀▀▀▀▀

   Это  подменю содержит два элемента для выбора тестируемого диска из
списка  дисков,  обнаруженных  программой,  а также для задания номера
диска  и  его  IDE-порта  вручную.  Используйте принудительное задание

           {"Mannualy set drive/port":DiskSelectManual}.

.topic DiskSelectList

 Select drive from list ▄     (F3)
  ▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀

   Выбор   диска  для  тестирования  из  списка  дисков,  обнаруженных
программой.  Для  IDE-дисков  выводится  название  модели  и  порт,  к
которому подключен диск.

   После  выбора  диска  информация  о  нем  отображается в окне. Если
программа  неправильно  определила  порт  диска,  то  вы можете задать
его вручную в меню {"Disk->Mannualy select drive/port":DiskSelectManual}

См. также: {"Меню 'Disk'":Disk}

.topic DiskSelectManual

 Manually set drive/port ▄     (Shift-F3)
  ▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀

   Выберите  номер  диска  для тестирования, а также, для IDE-дисков -
контроллер, к  которому  он  подключен, (Primary/Secondary)  и  способ
подключения (Master/Slave).

  Primary  контроллер  соответствует базовому адресу 1F0h, Secondary -
170h.

  Для тестирования не-IDE дисков выберите элемент 'Non-IDE drive'.

См. также: {"Select drive from list":DiskSelectList}
           {"Меню 'Disk'":Disk}

.topic Graphs

 Graphs ▄
  ▀▀▀▀▀▀▀

     Это  подменю  включает  в  себя  пункты  для  построения графиков
скоростей  чтения,  записи и верификации по отдельным дорожкам. Данные
графики   позволяют  увидеть  замещенные  дорожки,  зоны  с  различной
плотностью  записи  т.е.  дают более детальную информацию о скоростных
характеристиках диска и качестве его поверхности.

     Также данное подмею содержит график скорости чтения из кэш-буфера
винчестера,  в  зависимости  от  размера  блока.  Он позволяет увидеть
максимальную пропускную способность связки винчестер-контроллер-шина.

     Вы  можете  прервать  построение  любого графика, нажав <ESC> или
<Space>.  После  завершения построения графика по нажатию клавиши <F2>
можно записать его в файл в формате PCX.

См. также: {Linear Read Speed:GraphsRead}
           {Linear Write Speed:GraphsWrite}
           {Linear Verify Speed:GraphsVerify}
           {Cache Read Speed:GraphsCache}

.topic ConfigRPM

 Configuration - RPM Testing Method ▄
  ▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀

        RPM  (скорость  вращения диска в оборотах в минуту) может быть
определена  тремя  различными  способами.  Какие-то  методы  могут  не
работать  или  давать неверные результаты в силу каких-либо аппаратных
особенностей  диска/системы  в  которой  производится тестирование. По
умолчанию  HddSpeed  пытается  выбрать первый попавшийся метод, дающий
правдоподобный  результат.  Если выбор метода по умолчанию (Autodetect
Best Method) неудачен, то можно попробовать вручную задать желаемый.

        Метод  "Index"  заключается в подсчете числа сигналов Index от
жесткого  диска  за  определенный  промежуток  времени. Это достаточно
точный  метод, но не всегда применимый. Например, в новых ATA-3 дисках
данный сигнал, как правило, не используется.

        Метод  "Look-ahead  off  and read" означает отключение функции
опережающего  чтения  на  IDE  диске и посекторное считывание дорожки.
Очевидно,  что  для  считывания каждого следующего сектора потребуется
полный оборот диска, исходя из чего может быть подсчитан RPM.

     Метод   "Cache   overflow   and   write"   является  модификацией
предыдущего  метода,  но  в  нем  производится запись одного и того же
сектора,  причем  кэширование  НЕ отключается. Чтобы исключить влияние
буффера при записи, он сначала переполняется многократной записью. Это
единственный   из   трех  методов,  который  может  давать  правильные
результаты на SCSI дисках.

См. также: {Configuration:Config}
